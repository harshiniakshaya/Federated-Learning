SYSTEM OVERVIEW
This system simulates Federated Learning, where:
    Hospitals train models locally using their own data
    No raw data is shared 
    Only trained models are shared
    A central server combines these models into a global model

HOSPITAL A & HOSPITAL B (INDIVIDUAL SERVERS)
Step 1: Add Patient Data
    Patient records are stored in the hospital database (patient_records)
    Each hospital has its own dataset

Step 2: Train Local Model
    Process:
        kidney.csv 
            ↓
        Train initial model → initial_model.pkl  ✅ (base model)

        Hospital DB data
            ↓
        train_local()
            ↓
        hospitalA_model.pkl / hospitalB_model.pkl ✅ (updated model)
    
    Load initial_model.pkl (pretrained on kidney.csv)
    Fetch hospital data from database
    Clean + preprocess data
    Encode categorical values using saved encoders
    Train the model using hospital data
    Save updated model as:
    hospitalA_model.pkl
    hospitalB_model.pkl

Step 3: Push to Central
    Hospital sends its trained model to the central database
    Stored in respective tables:
        hospitala
        hospitalb
    Only model is shared (not patient data)

CENTRAL SERVER
Step 4: Run Aggregation
    This is the core of Federated Learning
    Aggregation Process:
        1. Connect to Central Database
            Access tables where hospitals uploaded models
        2. Fetch Latest Models
            Takes most recent model from each hospital
            [query: SELECT model_data FROM hospitala ORDER BY updated_at DESC LIMIT 1]
        3. Convert Binary → Model
            Stored models (BYTEA) are converted back using pickle.loads()
        4. Extract Model Parameters
            Each Logistic Regression model has:
                coef_ → weights (importance of each feature)
                intercept_ → bias
        5. Create Global Model: Replace old weights with averaged weights
            [global_assets = models[0]
             global_assets['model'].coef_ = avg_coef
             global_assets['model'].intercept_ = avg_intercept]
        6. Store Global Model
            Clear old model: TRUNCATE TABLE aggregated
            Store new model: INSERT INTO aggregated (model_data)
            NOTE: At any time, aggregated table contains latest global model only

Step 5: Pull from Central
    Hospitals download latest global model:
    aggregated → initial_model.pkl
    This becomes the new starting point

COMPLETE CYCLE
1. Train locally (hospital data)
2. Push model to central
3. Aggregate models
4. Pull global model
5. Train again
→ Repeat
